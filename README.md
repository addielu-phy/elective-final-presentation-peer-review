# 多元選修期末發表互評網站

個人發表用的即時互評平台：匯入學生姓名、抽籤決定上台順序、每人 1 分鐘報告、1 分鐘同儕評分與換場、學生掃 QR Code 評分、老師端即時收分並匯出 Excel。

## /goal

建立一個「多元選修期末發表」個人互評網站：

1. 老師可以匯入學生人名／座號。
2. 系統可以隨機抽籤產生個人上台順序。
3. 每位學生上台報告 1 分鐘。
4. 報告後進入 1 分鐘評分／換場時間，下一位可同時準備上台。
5. 同學掃 QR Code 進入評分頁，即時送分到老師端。
6. 針對學生掉線、重整、需要重掃 QR 的問題，加入固定 session、手機端身分記憶、自動重連與離線暫存補送。
7. 老師端可以查看即時平均、評分進度、未評分名單。
8. 活動結束可以匯出 Excel 與 JSON 備份。
9. 發布為 GitHub Pages 穩定網址。

## 評分規準

每項 1–10 分：

1. **內容重點清楚**：主題明確、重點聚焦、能在 1 分鐘內讓人聽懂。
2. **學習收穫／個人觀點**：能說出自己的收穫、觀察、反思或延伸想法。
3. **口語表達與時間掌握**：聲音清楚、節奏適當、時間控制佳。

同一位評分者對同一位報告者重複送出，會視為更新分數。

## 使用流程

1. 老師開啟公開網站。
2. 匯入學生名單：一行一人，例如 `01 王小明`。
3. 確認報告時間與評分／換場時間，預設都是 60 秒。
4. 按「儲存名單並抽籤」。
5. 按「建立即時收分 Session」。
6. 投影 QR Code 給學生掃。
7. 學生第一次進入後選擇「我是誰」。手機會記住這個活動與身分。
8. 老師依序按：
   - 開始報告 1:00
   - 評分／換場 1:00
   - 下一位＋開始報告
9. 活動結束下載 Excel / JSON。

## 掉線改善設計

這版針對「很多同學會掉線，需要重新掃 QR Code」做了幾個改善：

- **整場固定 QR Code**：不是每位報告者一個 QR，整場活動使用同一個 session URL。
- **固定 session code**：老師端會保存 session id，老師分頁重新整理後會優先使用同一個 id。
- **學生端記住活動連結**：學生手機 localStorage 會保存上次掃過的活動 URL。
- **學生端記住身分**：同一個活動內，學生重新整理後不用重新選姓名。
- **自動重連**：PeerJS/WebRTC 斷線後會自動嘗試重連。
- **離線暫存補送**：學生按送出時如果剛好離線，分數先存在手機，重連後自動補送。

## 重要限制

GitHub Pages 是穩定網站，不是雲端資料庫：

- 老師端分頁是即時收分主機，活動中必須保持開啟。
- 分數主要存在老師瀏覽器 localStorage；活動結束請務必匯出 Excel / JSON。
- 若學校網路阻擋 WebRTC 或 CDN，可能需要改成 Firebase / Google Apps Script / 本機 LAN server 版本。
